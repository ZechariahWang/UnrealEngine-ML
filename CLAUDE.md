- **Code style: plain "leetcode" style.** Write the simplest, most explicit code
  that works — no clever or fancy syntax. Stick to basic variables, `if`/`else`,
  `for`/`while` loops, plain functions and classes, and basic data structures
  (list / dict / set / tuple). Avoid: nested comprehensions, generator/lambda
  chains, decorators, metaclasses, walrus, complex `itertools`/`functools`
  pipelines, deep one-liners. A loop a beginner can read beats a one-line trick.
  Spell out intermediate steps into named variables instead of chaining. (This is
  about syntax legibility, not architecture — still don't add speculative
  abstractions, and keep the hardware calibration knobs.)
